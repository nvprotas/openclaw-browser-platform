# Tasty Coffee operational notes

- Начинай со страницы `https://shop.tastycoffee.ru/` и перед действиями закрывай cookie-баннер, если он перекрывает интерфейс.
- Основной поиск расположен в шапке (`header-search`); предпочтительно работать через него.
- Для открытия карточки товара используй только видимые ссылки из каталога/поиска с путём `/coffee/...` (заголовок карточки или её изображение), не извлекай slug из JSON/скриптов страницы.
- Не полагайся на клик по полному тексту названия товара: заголовок может быть разбит переносами (`<br>`), поэтому стабильнее кликать по ссылке карточки.
- На карточке товара до клика по покупке проверь, что выбран валидный вариант товара (например вес/помол), если сайт этого требует.
- Для `add_to_cart` сначала пробуй явные селекторы из pack, затем текстовые цели кнопок (`Купить`, `В корзину`).
- Считай добавление успешным только при изменении UI корзины: изменился счётчик, кнопка/состояние товара или появился переход в корзину.
- Для `open_cart` сначала используй кнопку корзины в хедере (`.header-cart__btn`), затем кнопку `Оформить заказ`; fallback-маршрут — `https://shop.tastycoffee.ru/basket`.
- Пустая корзина обычно распознаётся по текстам `Корзина` и `Корзина пуста`.
- Если после перехода к покупке появилась форма входа (`Войти`, `Электронная почта`, `Пароль`), фиксируй это как login-gate.
- Ассортимент и доступность могут зависеть от выбранного города/региона, поэтому одинаковые шаги могут давать разный результат.
- Останавливайся перед финальным подтверждением заказа, оплатой и другими необратимыми шагами.
